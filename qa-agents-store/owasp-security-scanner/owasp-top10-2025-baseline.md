# OWASP Top 10 (2025) — Cached Baseline

**Source of truth:** https://owasp.org/Top10/2025/ — always attempt a live `WebFetch` of this page first. Use this file only as a fallback when the live fetch fails (offline, blocked network, page unreachable), and tell the user the report is based on a cached snapshot rather than the live standard.

**Cached:** 2026-07-08. OWASP revises this list periodically — a live fetch may return categories, ordering, or wording that differs from below. If it does, the live fetch wins; update this file's cache date when that happens.

| Code | Category | Description | Static-analysis focus |
|---|---|---|---|
| A01:2025 | Broken Access Control | Unauthorized users can access restricted resources or perform actions beyond their permitted privileges. | Missing/inconsistent authorization checks on routes and handlers, IDOR (object references trusted from client input without ownership checks), privilege checks only on the client, default-allow authorization logic. |
| A02:2025 | Security Misconfiguration | Insecure default configurations, incomplete setups, or exposed security-sensitive settings. | Debug/verbose modes left on, permissive CORS (`*`), default credentials, exposed admin endpoints, missing security headers, overly permissive cloud/IaC config, directory listing enabled. |
| A03:2025 | Software Supply Chain Failures | Risks from compromised, outdated, or unverified dependencies, libraries, or third-party components. | Lockfile presence/integrity, dependencies with known CVEs, unpinned versions, install scripts from untrusted registries, missing SBOM/provenance checks in CI. |
| A04:2025 | Cryptographic Failures | Weak or misused cryptography that exposes sensitive data to unauthorized access or tampering. | Hardcoded secrets/keys, weak algorithms (MD5/SHA1 for passwords, ECB mode), missing TLS enforcement, plaintext storage/transmission of sensitive data, weak/predictable random number generation for security-sensitive values. |
| A05:2025 | Injection | Untrusted input inserted into commands or queries, letting attackers manipulate application behavior. | String-concatenated SQL/NoSQL queries, unsanitized input passed to `exec`/`eval`/shell calls, unescaped output in templates (XSS), unsafe deserialization, LDAP/XPath/command injection sinks. |
| A06:2025 | Insecure Design | Fundamental architecture/design flaws that omit adequate security controls from the start. | Missing rate limiting on sensitive operations, absent business-logic abuse protections, trust boundaries not enforced between components, security controls bolted on client-side only. |
| A07:2025 | Authentication Failures | Weaknesses in identity verification that let attackers bypass authentication or compromise accounts. | Weak password policy enforcement, missing brute-force/lockout protection, session tokens not invalidated on logout, predictable session IDs, missing MFA on sensitive flows where required. |
| A08:2025 | Software or Data Integrity Failures | Inability to verify authenticity/integrity of updates, components, or critical data. | Auto-update mechanisms without signature verification, CI/CD pipelines that pull unpinned/unverified artifacts, deserialization of untrusted data without integrity checks. |
| A09:2025 | Security Logging and Alerting Failures | Inadequate monitoring, logging, or alerting that prevents detection of and response to incidents. | Auth failures/access-control violations not logged, logs containing sensitive data (passwords, tokens, PII), no alerting hook for suspicious activity, logs that can be forged by user input (log injection). |
| A10:2025 | Mishandling of Exceptional Conditions | Improper error handling that exposes sensitive information or leaves systems in a vulnerable state on failure. | Stack traces/internal errors returned to clients, fail-open logic on exception (e.g. auth check that defaults to "allow" if it throws), unhandled exceptions leaving resources/locks in an inconsistent state. |
